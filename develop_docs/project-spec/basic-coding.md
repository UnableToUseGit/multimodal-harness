# 基础编码规范

## 代码风格

- 默认使用 Python 3.10+，统一使用 4 空格缩进。
- 源文件默认使用 UTF-8 编码；无明确需要时优先使用 ASCII 字符。
- 保持 import 清晰、稳定，避免无用 import 和循环依赖。
- 避免过长函数、过深嵌套和不必要的技巧性写法，优先保证可读性。

## 命名规范

- 模块、函数、变量使用 `snake_case`。
- 类使用 `PascalCase`。
- 常量使用 `UPPER_SNAKE_CASE`。
- 测试文件命名使用 `test_*.py`。

## 函数与类

- 函数应尽量职责单一、输入输出明确。
- 类名应准确表达职责，避免含义宽泛或过度抽象。
- 公共接口、核心数据结构和跨模块边界应尽量补充类型标注。

## 注释与日志

- 注释只解释设计意图、约束条件或不明显的实现原因，不重复代码表面含义。
- 正式代码中不要使用随意的 `print`，统一使用项目日志机制。
- 提交前应清理临时调试代码和无效注释。

### 路径处理                                                                                           
                                                                                                        
- 涉及文件系统路径时，内部实现统一使用 pathlib.Path。            
- 不应在核心逻辑中长期混用裸字符串路径和 Path 对象。                           
- 公共接口若需要接收路径输入，应显式声明为 Path 或 str | Path，并在进入核心逻辑后尽早转换为 Path。
