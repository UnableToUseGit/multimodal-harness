# 测试现象总结：
1. 对于本身结构性很好的视频而言（如 ufc_match, movie），分割规整，仅有几秒钟的误差，该误差目前被归因于采帧频率带来的误差。
2. 对于本身结构性不明显的视频 (如 lol match)，分割粒度较为混乱/不明晰，并且有关键事件发生中被切到了
3. 播客和电影的分割粒度设定的太细，以至于分割后的观感不佳（clip 太多）
4. caption 存在实体含义的分歧。比如，在 "podcast" 这个case 中，有些caption中，鲁豫是“采访者”，有些caption中则是“女性发言者 woman speaker”
5. 因为 title 是直接从 caption 中总结来的，可能会缺乏全局语境，例如 lol match case 中开头和结尾的片段。这两个片段内容是 BLG 战队的比赛/宣传画面，但在全局语境中其实是起到 intro 和 ending/transition 的作用，目前生成的 title 无法察觉这一点。
6. 目前的音频提取和转录是巨大的效率瓶颈
7. 为一个 1h 的视频生成 canonical atlas 生成大约需要 2 - 3 minute.


# 接下来该怎么做？

需要优化：
1. 提高验证力度: 
2. 优化音频 ASR 的效率
3. 优化 caption & title 
4. 优化分割效果

具体怎么做：
第一个优化： 继续扩充 case 数量（视频类型不要扩充太多）; 把 segmentor 检测分割点的 boundary rational 保存下来用于分析
第二个： 还需要调研
第三个： 提供全局信息
第四个：

首先我心中预期的分割效果:
1. 切的不要太碎。
2. 尽可能贴近该视频的某个内在语义结构。（因为语义结构可能很多层次，并且结构也没那么清晰，所以言之有理即可）
3. 不要切到某个关键事件之中，导致事件语义被破坏。比如一场团战正在进行中被切断了。

现在有两种思路
- 提供 segmentation proposal 给分割模型: 人工预定义 或 planner 给出针对特定视频（非常细粒度的类别，比如英雄联盟比赛）的 segmentation proposal，直接告诉 segmentor 这个视频应该分为哪几种阶段
- 提供 segmentation policy 给分割模型: 人工预定义 或 planner 给出针对一大类视频（较粗力度，比如电竞比赛视频）的 segmentation policy，告诉 segmentor 关注怎样粒度的内在结构，什么要切，什么不要切。

两者的关键区别就在于，“分割成哪几种阶段” 这个决策是谁的职责？  第一种方案是 planner/人工预定义来负责，第二种则是 segmentor 在policy指导下自己决定。
1. 这个职责如果是人工预定义的，显而易见的问题就是成本高，每种游戏，每种影视剧/综艺/相声 都要人工预定义，很夸张。
2. 如果让 planner 来负责，对于很长的视频（> 30min），planner 只能有限采帧，它必须要结合 pretrain knowledge 来给出proposal，这是否可靠？
3. 如果让 segmentor 来负责，现在的测试结果并不太好，其暴露出的问题能否通过优化 policy 的内容得以解决
