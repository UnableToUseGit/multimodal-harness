# Task Derivation Agent

## Purpose: 

Given a user task description and a canonical atlas, this module derivates a newly task-oriented atlas. This atlas provide necessary information required by tasks.

## Responsibilities: 

- Accept task description and analyze what information does it requrie?
- Collect `AtlasSegment` from `CanonicalAtlas` contained the required information and process them into `DerivedAtlas`.
- Write the `DerivedAtlas` into `workspace`.

## Inputs:

- `task_request` (`str`, required): A user request that describs what he want to do with a video.
- `canonical_atlas` (`CanonicalAtlas`, required): a domain model represent the video cannonical atals generated by Canonical Atlas Agent.
- `workspace` (`BaseWorkspace`, required): the workspace used for contain generated video atlas.
- `planner` (`BaseGenerator`, required): component participates in producing canonical atlast construction plan.
- `segmentor` (`BaseGenerator`, required): component used for boundary detection
- `captioner` (`BaseGenerator`, required): component used for local segment caption and global text information generation.

## Outputs:

- `DerivedAtlas` (`object`): a domain model represent an atlas derived from canonical atlas given the `task_request`.
- A file directory (`workspace`) correpsonding to `DerivedAtlas`
- `DerivationResultInfo` (`object`): a domain model contain statistics or summary of derivation.

## Dependencies:

- `CanonicalAtlasAgent`: Its output `CanonicalAtlas` is the target for `DerivedAtlasAgent` to process.
- `Generator`: A module that provides a lightweight interface for LLM interactions. The `planner`, `segmentor`, and `captioner` components are derived from this module."
- `Workspace`: A module that provides a execution environment for agent. It executes the action generated by agent in file system.

## Internal Workflow:

1. **Selection**
Content: This module firstly use `planner` to read all `AtlasSegment` and `task_request` to select those that contain task-required information. And generate a derivation policy for each one.

2. **Derivation**
Content: Then this module will derive task-aware atlas from each selected `AtlasSegment` following its assigned derivation policy. Derivation process contains three operation: 
- Re-grounding: raw video clip from `CanonicalAtlas` is usually coarse-grained. In most times, we need to localize sub-video-clip properly meet task required.
- Part Subtitles Extraction: after re-grounding, only keep subtitles in the sub-video-clip is necessary.
- Re-caption: generate detailed caption for the new sub-video-clip and other important text information.
Notably, derivation process for different `AtlasSegment` can be executed in parallel.
Critical Output:
- A list of derived `AtlasSegment`
- `DerivationResultInfo` 

3. **Atlas Assembly**
Take all derived `AtlasSegment` together and generate:
- a global title
- a global abstract
- final canonical segment titles

4. **Workspace Finalization**
Validate the workspace structure and move intermediate artifacts such as `EXECUTION_PLAN.json` into `.agentignore/`.
