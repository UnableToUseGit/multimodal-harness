# Derived Atlas Agent

## Purpose

`DerivedAtlasAgent` creates a task-aware atlas from an existing canonical atlas. The derived atlas keeps only the source segments that matter for the requested task, re-grounds each selected segment to a tighter sub-clip, regenerates task-oriented text, and writes a new derived workspace for browsing and review.

## Responsibilities

- accept a `task_request` and a `CanonicalAtlas`
- select relevant canonical `AtlasSegment` items and assign a `DerivationPolicy` to each selected segment
- refine each selected segment to a task-specific sub-range
- prune subtitles to the refined range
- regenerate task-oriented title, summary, and detailed caption
- aggregate derived-segment statistics into a `DerivedAtlas`
- write the derived workspace plus derivation metadata files

## Inputs

- `task_request` (`str`, required): the user task that defines what information should be preserved
- `canonical_atlas` (`CanonicalAtlas`, required): the canonical atlas used as source material
- `planner` (`BaseGenerator`, required): selects source segments and produces `DerivationPolicy`
- `segmentor` (`BaseGenerator`, required): refines each selected source segment to a tighter time range
- `captioner` (`BaseGenerator`, required): generates derived title, summary, and detailed caption
- `workspace` (`BaseWorkspace`, required): destination workspace for the derived atlas
- `num_workers` (`int`, optional): parallel worker count for per-segment derivation

For local runs, `scripts/run_task_derivation.py` loads `CanonicalAtlas` from a canonical workspace directory through `video_atlas.agents.task_derivation.load_canonical_workspace`.

## Outputs

- `DerivedAtlas`: final task-aware atlas domain object
- `DerivationResultInfo`: derivation metadata used for review and debugging
- derived workspace files:
  - `README.md`
  - `TASK.md`
  - `derivation.json`
  - `segments/<derived-segment>/README.md`
  - `segments/<derived-segment>/video_clip.mp4`
  - `segments/<derived-segment>/SUBTITLES.md` when available
  - `.agentignore/DERIVATION_RESULT.json`

## Internal Workflow

1. **CandidateGeneration**

`planner` reads the task request plus the canonical segment summaries/captions and returns the subset of source segments that matter for the task. Each selected source segment receives a `DerivationPolicy`:

- `intent`
- `grounding_instruction`

2. **Derivation**

Each selected canonical segment is processed independently and can run in parallel:

- **Re-grounding**: `segmentor` produces a tighter start/end range inside the source segment
- **Clip Extraction**: the derived segment is re-cut from the canonical source video using the refined range
- **Subtitle Pruning**: only subtitle lines fully inside the refined range are kept
- **Re-caption**: `captioner` generates task-oriented `title`, `summary`, and `caption`

Each successful derived segment records:

- its derived segment id
- the source canonical segment id
- the `DerivationPolicy` that justified the derivation

3. **Aggregation**

Aggregation is generated by code, not by an extra LLM call. The agent computes:

- total derived segment count
- total derived duration
- average derived duration
- a per-derived-segment breakdown containing intent and precise start/end times

These values are packed into `DerivedAtlas.global_summary` and `DerivedAtlas.detailed_breakdown`.

4. **Workspace Finalization**

The agent writes the final derived workspace and stores derivation metadata for later review:

- public files in the workspace root and `segments/`
- `derivation.json` for review tooling
- `.agentignore/DERIVATION_RESULT.json` for detailed derivation bookkeeping

## Derived Workspace Contract

Each derived segment README uses these fields:

- `DerivedSegID`
- `SourceSegID`
- `Start Time`
- `End Time`
- `Duration`
- `Title`
- `Summary`
- `Detail Description`
- `Intent`

`derivation.json` contains:

- `task_request`
- `global_summary`
- `detailed_breakdown`
- `derived_segment_count`
- `source_canonical_atlas_path`

## Constraints / Notes

- derivation is intentionally task-reductive, not coverage-preserving like the canonical pipeline
- re-grounding is constrained to the time span of each selected source canonical segment
- clip extraction requires `CanonicalAtlas.source_video_path`
- parallelism is applied at the per-selected-segment derivation stage
