# 播客转小红书笔记

根据用户指定的播客，创作小红书笔记，并可选择发布到小红书。

因为播客的内容量较大，所以这类笔记一般将笔记内容渲染成一张张的图片进行呈现。

---

## 工作流程

### 第一步：撰写笔记

#### 1. 笔记内容指导

先新建一个 Markdown 文件，用于撰写笔记内容。

**优先从播客中提炼：**

- 播客主题与核心观点
- 最值得传播的关键判断
- 嘉宾或主持人的典型原话
- 必要的背景、例子与解释

**笔记内容应：**

- 以自然段为主，而不是关键词堆砌
- 使用小标题组织结构
- 适当引用嘉宾或主持人的原话，提高可信度和“播客原声感”
- 适度使用加粗、斜体等方式强调重点句

**关于笔记的封面图：**

笔记的封面图应优先使用**真实的视频播客画面**。

封面图的选择优先级如下：

1. 优先使用视频播客中的高质量画面  
   例如嘉宾的清晰特写、构图稳定的双人对谈画面、能够体现播客氛围的场景画面。

2. 如果没有合适的视频画面，则使用播客的 thumbnail  
   例如 YouTube 缩略图、节目官方封面图等。

3. 如果没有视频画面，也没有合适的 thumbnail，则使用与播客主题相关的外部图片  
   例如公开可用的概念配图、人物相关图片、主题相关视觉素材。

封面图应满足以下要求：

- 清晰，分辨率足够高
- 与播客主题强相关
- 适合做首图，具有一定视觉吸引力
- 不要包含杂乱水印、过密小字或明显失真的画面

选好封面图后，应将图片路径写在 Markdown 文件开头的 YAML frontmatter 中，供渲染脚本使用。


#### 2. 笔记文件的结构要求

撰写笔记的 Markdown 文件分成两个部分：

1. 封面页
2. 正文主体

两者之间使用一个 `---` 分隔进行分页。

其中：

- 封面页用于展示笔记的标题，封面图，以及针对该播客的简要介绍（说明这期播客讲了什么、为什么值得看、本文将提炼哪些重点）
- 正文主体是完整连续的文章内容
- 正文主体内部不再进行手动分页

该 markdown file 结构如下：

```markdown
---
title: "笔记标题"
source: "播客来源"
intro_image: "./selected-frame.jpg"
---

"放在封面页的文本内容，这是读者最先看到的内容，需要向观众说明这期播客是什么，他为什么值得你观看。这一部分内容的撰写不需要加任何标题。"

---

正文……

> 嘉宾原话……

正文……
```

说明：

- 文件开头可以包含 YAML frontmatter，用于记录全局元数据，例如标题，封面图路径等
- YAML frontmatter 不属于正文内容
- YAML frontmatter 之后，先写封面页的文本内容（针对该播客的简要介绍）
- 封面页之后，用一个 `---` 分隔正文主体
- frontmatter 中应有 `intro_image`，渲染器会默认将该图片作为封面图
- 正文主体内部由渲染脚本自动分页，不再通过 `---` 手动切页

---

### 第二步：渲染图片

渲染后的正文图片不应做成海报式卡片，也不应做成一页只放几个 bullet 的摘要卡片。

更合适的形式是：

- 按文章方式连续撰写正文
- 用图片形式分页呈现
- 每页适合手机阅读
- 整体看起来像“小红书图文化长文”，而不是 PPT 卡片

#### 3. 使用专用渲染脚本

笔记的 markdown 写好后，使用本 skill 自带的专用渲染脚本生成图片：

```bash
python scripts/render_xhs.py <markdown_file> [options]
```

该脚本会：

- 单独渲染封面页
- 将正文主体转为适合小红书阅读的长文图片
- 按固定页面宽高自动分页
- 输出多张正文图片，例如：
  - `intro.png`
  - `page_1.png`
  - `page_2.png`
  - `page_3.png`
  - ...

渲染目标不是生成“多主题卡片”，而是生成一组风格统一、适合手机阅读的笔记图片。

---

### 第三步：准备帖子标题和笔记描述

通过前两个步骤我们已经获得了笔记内容的渲染图片，接下来我们还需要撰写一个帖子标题和笔记描述来用于最后的发布。

**帖子标题**：不超过 20 字，吸引眼球，可用数字/疑问句/感叹号增强吸引力。

**笔记描述**：可以看做是该播客小红书笔记的高度概括版，段落清晰，点缀少量 Emoji（每段 1-2 个），短句短段，结尾附 5-10 个 SEO 标签。

---

### 第四步：发布小红书笔记（可选）

**前置条件**：配置好 `.env` 文件中的 `XHS_COOKIE`（详见 `references/params.md`）

```bash
# 默认仅自己可见（推荐先预览确认）
python scripts/publish_xhs.py --title "帖子标题" --desc "笔记描述" \
  --images cover.png card_1.png card_2.png

# 确认无误后公开发布
python scripts/publish_xhs.py --title "帖子标题" --desc "笔记描述" \
  --images cover.png card_1.png card_2.png --public
```

> **默认以「仅自己可见」发布**，加 `--public` 参数才会公开。
