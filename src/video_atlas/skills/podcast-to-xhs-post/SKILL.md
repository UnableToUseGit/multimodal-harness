# 播客转小红书笔记

根据用户指定的播客，创作小红书笔记，并可选择发布到小红书。

小红书笔记分为图片，标题，正文三个部分。

小红书笔记的正文部分最多写1000个字符，而播客的内容量往往较大，所以这类笔记一般将真正的正文内容制作成图片的形式，而笔记的正文部分填写相对概括性的信息。

---

## 工作流程

### 第一步：撰写笔记标题和正文部分

根据用户需求和播客内容，创作符合小红书风格的标题和正文：

**标题**：不超过 20 字，吸引眼球，可用数字/疑问句/感叹号增强吸引力。

**正文**：段落清晰，点缀少量 Emoji（每段 1-2 个），短句短段，结尾附 5-10 个 SEO 标签。

---

### 第二步：制作笔记的封面

共有两种笔记封面制作方法，取决于用户指定的播客是否是视频的形式。

如果是视频播客，则可以从视频中寻找合适的帧画面作为笔记的封面，例如被采访人物的特写镜头，但要保证这张画面的清晰度，美感。

如果不是，则可使用以下方式渲染一张图片作为封面：

首先创建一个 Markdown File 用于做封面图片的渲染。

```markdown
---
emoji: "主题相关的emoji表情"
title: "封面大标题（≤15字）"
subtitle: "封面副标题（≤15字）"
---
```

然后使用 `scripts/render_cover_xhs.py` 进行渲染：

```bash
python scripts/render_cover_xhs.py <markdown_file> [options]
```

常用示例：

```bash
# 默认（sketch 主题）
python scripts/render_cover_xhs.py content.md

# 切换主题
python scripts/render_cover_xhs.py content.md -t playful-geometric
```

生成结果：`cover.png`（封面）

**可用主题**（`-t`）：`sketch`、`default`、`playful-geometric`、`neo-brutalism`、`botanical`、`professional`、`retro`、`terminal`

> 完整参数说明见 `references/params.md`

### 第三步：制作笔记的正文图片

正如前文所言，正文部分填写的是概括性的内容，真正详细的正文应该制作成图片。

这些正文图片不应做成海报式卡片，也不应做成一页只放几个 bullet 的摘要卡片。

更合适的形式是：

- 按文章方式连续撰写正文
- 用图片形式分页呈现
- 每页适合手机阅读
- 整体看起来像“小红书图文化长文”，而不是 PPT 卡片

#### 1. 先写完整正文，而不是按单张图分别构思

先新建一个 Markdown 文件，用于撰写正文图片内容。

优先从播客中提炼：

- 播客主题与核心观点
- 最值得传播的关键判断
- 嘉宾或主持人的典型原话
- 必要的背景、例子与解释

正文内容应：

- 以自然段为主，而不是关键词堆砌
- 使用小标题组织结构
- 适当引用嘉宾或主持人的原话，提高可信度和“播客原声感”
- 适度使用加粗、斜体等方式强调重点句

正文整体应足以支持最终生成约 `8-12` 张正文图片。

#### 2. 引导页和正文主体之间用 `---` 分隔

Markdown 文件分成两个部分：

1. 引导页
2. 正文主体

两者之间使用一个 `---` 分隔。

其中：

- 引导页用于说明这期播客讲了什么、为什么值得看、本文将提炼哪些重点
- 正文主体是完整连续的文章内容
- 正文主体内部不再手动分页

推荐写法如下：

```markdown
---
title: "笔记标题"
---

# 为什么这期播客值得看

这期播客主要讨论了……
我整理出了最值得分享的几个重点。

---

# 第一个关键判断

正文……

## 为什么这很重要

正文……

> 嘉宾原话……

正文……
```

说明：

- 文件开头可以包含 YAML frontmatter，用于记录全局元数据，例如标题等
- YAML frontmatter 不属于正文内容
- YAML frontmatter 之后，先写引导页
- 引导页之后，用一个 `---` 分隔正文主体
- 正文主体内部由渲染脚本自动分页，不再通过 `---` 手动切页

#### 3. 使用专用渲染脚本自动分页

正文 markdown 写好后，使用本 skill 自带的专用渲染脚本生成图片：

```bash
python scripts/render_body_xhs.py <markdown_file> [options]
```

该脚本会：

- 单独渲染引导页
- 将正文主体转为适合小红书阅读的长文图片
- 按固定页面宽高自动分页
- 输出多张正文图片，例如：
  - `intro.png`
  - `page_1.png`
  - `page_2.png`
  - `page_3.png`
  - ...

渲染目标不是生成“多主题卡片”，而是生成一组风格统一、适合手机阅读的正文图片。

#### 4. 关于配图

正文图片中可以插入少量配图，但这不是必须项。

只有在下面情况时才建议加入图片：

- 这是视频播客，且某一帧画面能明显增强理解或真实感
- 该图片与当前正文内容强相关
- 图片本身足够清晰，适合小红书阅读

不要为了让页面“更丰富”而机械插图。大多数情况下，正文图片应以文字为主。

### 第四步：发布小红书笔记（可选）

**前置条件**：配置好 `.env` 文件中的 `XHS_COOKIE`（详见 `references/params.md`）

```bash
# 默认仅自己可见（推荐先预览确认）
python scripts/publish_xhs.py --title "笔记标题" --desc "笔记描述" \
  --images cover.png card_1.png card_2.png

# 确认无误后公开发布
python scripts/publish_xhs.py --title "笔记标题" --desc "笔记描述" \
  --images cover.png card_1.png card_2.png --public
```

> **默认以「仅自己可见」发布**，加 `--public` 参数才会公开。
